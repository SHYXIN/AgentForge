# 前端 UI 文本规范

## 概述

本规范旨在确保 AgentForge 前端项目中的所有 UI 文本都使用纯文本，不使用任何 emoji 字符，以保持专业、简洁的用户界面。

## 适用范围

适用于所有前端文件，包括但不限于：
- React 组件（.tsx, .jsx）
- Vue 组件（.vue）
- HTML 模板（.html）
- TypeScript/JavaScript 文件（.ts, .js）
- CSS 内容（CSS 中的 content 属性）
- 配置文件中的 UI 文本

## 规则

### 1. 禁止使用 Emoji

**❌ 错误示例：**
```tsx
<button>保存 ✅</button>
<span>状态：完成 🎉</span>
<input placeholder="搜索 🔍" />
```

**✅ 正确示例：**
```tsx
<button>保存</button>
<span>状态：完成</span>
<input placeholder="搜索" />
```

### 2. 使用纯文本替代方案

| 用途 | ❌ 避免 | ✅ 推荐 |
|------|---------|---------|
| 成功状态 | ✅ 成功 | 成功 |
| 错误状态 | ❌ 错误 | 错误 |
| 警告信息 | ⚠️ 警告 | 警告 |
| 加载状态 | ⏳ 加载中 | 加载中 |
| 完成状态 | 🎉 完成 | 完成 |
| 提示信息 | 💡 提示 | 提示 |
| 重要信息 | ⭐ 重要 | 重要 |
| 新功能 | 🆕 新功能 | 新功能 |
| 删除操作 | 🗑️ 删除 | 删除 |
| 编辑操作 | ✏️ 编辑 | 编辑 |
| 查看详情 | 🔍 查看 | 查看 |
| 刷新操作 | 🔄 刷新 | 刷新 |
| 设置选项 | ⚙️ 设置 | 设置 |
| 用户信息 | 👤 用户 | 用户 |
| 消息通知 | 📧 消息 | 消息 |
| 文件操作 | 📁 文件 | 文件 |
| 帮助信息 | ❓ 帮助 | 帮助 |

### 3. 标点符号使用

- 使用中文标点符号：（）、；：""''
- 避免使用英文标点作为装饰
- 不要使用特殊符号作为列表装饰（如 →、•、· 等）

**❌ 错误示例：**
```tsx
<li>→ 第一项</li>
<li>• 第二项</li>
```

**✅ 正确示例：**
```tsx
<li>第一项</li>
<li>第二项</li>
```

### 4. 按钮文本规范

按钮文本应简洁明了，使用动词或动词短语：

**✅ 推荐：**
- "提交"
- "取消"
- "确认"
- "返回"
- "下一步"
- "保存"
- "删除"
- "编辑"

### 5. 状态指示

使用文字描述状态，不使用图标或 emoji：

**✅ 推荐：**
- "加载中..."
- "保存中..."
- "处理完成"
- "连接失败"
- "正在验证..."

### 6. 错误提示

错误提示应使用清晰的文字描述：

**❌ 错误示例：**
```
❌ 用户名不能为空
⚠️ 密码长度不足
```

**✅ 正确示例：**
```
用户名不能为空
密码长度不足
```

## 检查清单

在提交代码前，请检查：

- [ ] 所有按钮文本不包含 emoji
- [ ] 所有标签文本不包含 emoji
- [ ] 所有提示信息不包含 emoji
- [ ] 所有错误消息不包含 emoji
- [ ] 所有状态指示不包含 emoji
- [ ] 所有占位符文本不包含 emoji
- [ ] 所有标题文本不包含 emoji
- [ ] 所有工具提示不包含 emoji

## 代码审查要点

代码审查时，请特别注意：
1. 搜索常见 emoji 字符（如 ✅、❌、⚠️、💡、🎉 等）
2. 检查是否有使用 Unicode 表情符号
3. 确认所有用户可见文本都符合本规范

## 工具和自动化

建议在项目中添加以下自动化检查：
- ESLint 规则：禁止在字符串中使用 emoji 字符
- Prettier 配置：确保代码格式一致
- 预提交钩子：自动检查新增代码中的 emoji
